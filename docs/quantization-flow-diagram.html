Canary Rollout Logic
How 5% canary rollout works
# Deterministic — same device always same result for same version hash = SHA256(device_id + ":" + version) bucket = int(hash[:4], big-endian) % 100 in_cohort = bucket < canary_pct # e.g. 5 # canary_pct=5 → 5% of devices get update # canary_pct=100 → everyone gets update # canary_pct=0 → nobody gets update (emergency block)
canary_pct = 5 → only 5 of 100 devices see the update
5% — update
95% — wait
canary_pct=5 → start → 25 after 1 day → 50 after 3 days → 100 full rollout
